## Objective

Chunk the regulation PDFs on their document hierarchy (chapter → paragraph →
sub-clause), each chunk carrying `heading_path`, `clause_id`, `version`,
`effective_date`, and source-location metadata. Chunks are the retrieval unit
for T3.2 and the citation target for findings. **Clause-boundary fidelity is
the invariant** — a chunk must never straddle two clause_ids that appear in
`clauses.jsonl`.

## Chunk model (`rag/schemas.py`)

```python
class RegulationChunk(BaseModel):
    model_config = ConfigDict(extra="forbid")
    chunk_id: str                 # f"{doc_slug}::{clause_id or f'front-{n}'}"
    doc: str                      # matches clauses.jsonl clause.doc exactly
    heading_path: list[str]       # e.g. ["Chapter III — Conduct", "11. Activation", "(6)"]
    clause_id: str | None         # canonical form (grammar below); None for
                                  # front matter / annex text with no clause number
    version: str                  # from sources/MANIFEST.json per document
    effective_date: datetime.date # from sources/MANIFEST.json per document
    text: str
    page_start: int
    page_end: int
    char_span: tuple[int, int]    # offsets into the loader's full-document text
```

This **composes** the frozen contract (same `doc`/`clause_id`/`version`/
`effective_date` vocabulary as `RegulationClause`) without touching it — not a
CONTRACT CHANGE.

## Canonical clause_id grammar

Must byte-match Track B's records: `<int>` optionally followed by
parenthesized sub-parts, no spaces — e.g. `11(6)`, `7(b)`, `6(a)(vi)`.
Normalizer: strip whitespace inside references, lowercase alpha sub-parts,
roman numerals kept lowercase. The reconciliation gate (Gate A) is the
enforcement; if B's form and the chunker's form disagree, normalize the
**chunker** to B's — `clauses.jsonl` is the reference vocabulary.

## Gates

**Gate A — reconciliation join (mechanical, blocking):** for every record in
`clauses.jsonl`, there exists **exactly one** chunk with matching
(`doc`, `clause_id`), and the clause's `text` is a semantic subset of the
chunk's `text` (assert: ≥80% of the clause text's content words appear in the
chunk). 19/19 must pass.

**Gate B — golden boundaries (hand-checked, two-step):** the chunker emits
`tests/fixtures/chunks/anchor-boundary-report.json`: for the anchor PDF, every
chunk's (`clause_id`, `heading_path`, `page_start`, first 60 chars, last 60
chars). Claude Code commits this report with tests marked
`@pytest.mark.xfail(reason="pending hand-check")` for the golden comparison.
**The hand-check happens at review in the Track C chat** (clone the branch,
verify 10 boundaries against the pinned PDF, including at least: one chapter
start, one multi-page paragraph, one deep sub-clause like `6(a)(vi)`-style,
and the para-90 KYC bridge). On confirmation the xfail is lifted and the 10
verified boundaries become the golden fixture. T3.1 is **done** only after
this promotion — mirror of T1.1's hand-verified-nesting protocol.

**Gate C — determinism:** two consecutive runs over the same PDFs produce
byte-identical chunk output (stable ordering, stable chunk_ids).

## Implementation notes

- PDF text extraction: `pdfplumber` (add to pyproject deps). RBI Master
  Directions are born-digital text PDFs; no OCR path. Preserve page numbers.
- Hierarchy detection: regex over numbering patterns anchored at line starts
  (`^\d+\.`, `^\(\w+\)`, `^Chapter [IVXLC]+`), stack-based path tracking.
  Headers/footers (page furniture, "RBI/2025-26/…" banners) stripped by
  repetition detection across pages, not by hardcoded strings.
- Front matter, definitions, annexes: chunked with `clause_id=None` and an
  honest heading_path — they are retrieval-relevant even when unnumbered.
- No embedding, no indexing, no retrieval here — that is T3.2. This task ends
  at structured chunks.
