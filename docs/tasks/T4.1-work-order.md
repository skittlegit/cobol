## Objective

Run a system under test over the frozen `v1-pre` benchmark without exposing
gold labels to the system, persist replayable trajectories, and prove the
three-track seam: a Track B mutation is investigated through Track A's real
tool layer and consumed by Track C evaluation.

## Inputs

- Headline split: `data/benchmark/v1-pre/test.jsonl` (204 rows, T2.7 revision).
- Capacity catalogue: `data/benchmark/drift_instances.plausible.jsonl`
  (562 accepted rows).
- Seed sources: `data/benchmark/seed/programs/`.
- System model: a live `DecisionModel` adapter. Cached fixture responses may
  exercise gates but must never be called an M4 headline run.
- Tools: `RealToolLayer`, injected through the frozen `ToolLayer` Protocol.

## Deliverables

1. `src/cobol_archaeologist/eval/schemas.py`: non-contract evaluation records.
2. `src/cobol_archaeologist/eval/materialize.py`: reconstruct synthetic source
   from immutable base + recorded mutation, with an exact target-edit check.
3. `src/cobol_archaeologist/eval/run.py`: resumable runner and manifest.
4. `src/cobol_archaeologist/model/provider.py`: live provider adapter behind
   `DecisionModel`; credentials are environment-only and never persisted.
5. `tests/test_eval_run.py` and `tests/fixtures/eval/`.

## Integrity rules

- The runner may pass clause text, source scope, and tool observations to the
  system. It must not pass `drift_type`, labels, gold rationale, mutation name,
  old/new edit values, or instance ID.
- Source reconstruction is fail-closed. The requested old fragment must occur
  at the recorded locus before replacement; ambiguity or missing source is an
  explicit unavailable record, not an abstention scored as model behavior.
- Every attempted row has a trajectory or a typed infrastructure failure.
- Resume keys include instance ID, source hash, model ID, budgets, prompt
  version, tool version, and repository commit.
- Outputs are written under `data/eval/m4/` and are not benchmark inputs.

## Gates

1. **Week-7 three-track seam:** generate one mutation through Track B's public
   mutator, materialize it, let an agent call at least one real Track A tool,
   and score the resulting Track C evaluation record.
2. Gold-hidden test: a spy provider sees none of the prohibited fields.
3. Real/stub constructor swap requires no runner code change.
4. Mutation materialization covers local, copybook, deletion, and
   interprogram edits and rejects ambiguous replacements.
5. Resume is deterministic and never duplicates a completed instance.
6. Missing provider credentials/source/compiler fail closed with a manifest
   reason; they never produce headline numbers.
7. Offline gates use cached responses; the live 204-row run is an explicit
   network gate.
