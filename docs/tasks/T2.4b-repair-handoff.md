met every floor by replication. Nothing forbade duplicates, so duplicates
counted — *absence read as permission*, the same defect pattern as BL-15/BL-17,
now sitting under the benchmark's headline claim.

## 4. The four shape defects (judge's reasons are substantively correct)

Verbatim reasons are in `judgements.sample50.jsonl`.

**(a) D2 / MO-2 / KYCSYNC3 — orphaned dataflow.** 20 rows.
`IF WS-TODAY-DAY > WS-DUE-DAY MOVE 'OVERDUE' ... ELSE MOVE 'OK' ... END-IF.`
→ `MOVE 'OK' TO WS-STATUS.`
Luna: *"ignores both WS-TODAY-DAY and the computed WS-DUE-DAY, making the
preceding date logic meaningless."* Correct: `1000-MAIN` still computes
`WS-DUE-DAY = WS-RECEIPT-DAY + 7` and nothing reads it. No maintainer leaves a
computation feeding nothing. The `_apply_mo2` ELSE-path "keep the ordinary
fallback" design (`mutate.py:1057-1069`) produces an unconditional success stub
whenever the fallback *is* the success case.

**(b) D3 / MO-3 / LATEFEE1+LATEFEE2 — mechanical AND→OR.** ~14 rows.
Luna: *"a direct, simplistic regulatory-violating Boolean mutation rather than
credible legacy drift."*
**This one is ours, and the test enshrines it.** After the judge rejected
comparator inversions (`>` → `<=`) as artificial, MO-3 was rewritten to emit
*connective* inversions (`AND` → `OR`) and the result was named "grace widening."
Same one-token mechanical flip, different token. **`tests/test_mutate.py:660`
asserts `OR {disjunct} > ZERO`** — the gate forbade comparator inversions, so
connective inversions were permitted by omission. Fix the operator *and* the
test.
`_apply_mo3_grace_widening` `mutate.py:1116`.
The `_apply_mo3_capitalization` shape (INTCOMP1) **passed** — keep it.

**(c) D3 / MO-3× / TRNVAL1 + OVRLIM1 — internal incoherence.** ~14 rows.
TRNVAL1: `WS-FAIL-REASON` initialized to zero and never changed → validation is
vacuous, posting always occurs. OVRLIM1: consent flag never reset → stale
consent authorizes later over-limit transactions. Luna calls both "suspiciously
incomplete." These are *broken* programs, not *drifted* ones.
`_apply_mo3x` `mutate.py:1596`.

**(d) D1 / MO-1 / KYCSCHED2 — `stale_source=grid_fallback` with no story.** 1 row.
High-risk KYC interval 2→5 while medium/low tiers keep exact regulatory values.
The grid had no real prior for the leaf, so it picked an arbitrary laxer number.
Luna: *"The arbitrary single-literal deviation looks more like an artificial
mutation."* Lowest priority — D1 is at 87.5% and is not gate-critical.

**Marginality, not brokenness:** the *same* KYCSYNC3 semantic drift, restyled,
came back **2 plausible / 3 implausible**. Luna is **style-sensitive**. A
robustly plausible shape would not flip on cosmetics — that is the bar to author
against, and it means a shape that merely *sometimes* passes is not repaired.

legitimate — and Luna's reasons are technically correct in all four cases. Keep
that distinction; it is the difference between repairing and gaming.

## 7. Gotchas a fresh agent will otherwise hit

- **BL-16 is the law.** *Local harnesses generate hypotheses; only the real
  toolchain closes gates.* A pure-text probe once reproduced a build's Gate E
  number exactly, was trusted through six iterations, then diverged (0.5457 vs
  the real 0.6537) because it had no validation step and could not model rejected
  mutations changing the sample. **Gate E's and the judge gate's number of record
  is the WSL run.** This session it bit again: the simulation predicted
  per-feature `diff_size` 0.534 / `touched_line_count` 0.540; the real build
  returned **0.618 / 0.642**.
- **Environment split:** `cobc` and `tree-sitter` are **both unavailable on the
  Windows host** (`WinError 193`). Builds and the full suite run **only in WSL**
  at `/mnt/c/Users/deepa/Github/cobol`. Pure-JSON/git tests (e.g. `test_bl8_*`,
  `test_bl12_*`) do run on Windows.
- **Windows `open()` defaults to cp1252.** `json.load(open(path))` mojibakes the
  canonical `×` (U+00D7) in `MO-1×` operator keys into `Ã—`. Always pass
  `encoding='utf-8'`. The manifest is *not* corrupt — this bug was hit and
  falsely reported once this session.
- **`schemas.py` is Track C's and frozen.** `CurrentValue` is `extra="forbid"` —
  per-clause knobs like `mo1_mode` live in the free-form `check` dict, never in
  `current_value`. Touching `schemas.py`/`tools.py`/`docs/CONTRACT.md` is a
  CONTRACT CHANGE: **stop and ask.**
- **Ownership:** Track B owns `benchmark/`, `data/`. Never edit another track's
  modules.
- **Line-number fidelity is sacred** (CLAUDE.md #4). Benchmark labels are
  line-level.
- **`ClauseDataError` is deliberately NOT a `MutationRejected`** — the build
  catches the latter and turns it into a statistic. That distinction is exactly
  how the MO-2 compile bug hid for so long.
- **`slice_on` is a BACKWARD slice** (what affects a var), not forward/use. A
  coupling fix built on it silently did nothing once already.
- **Do not filter `d1` in `build.py`** — `pair_candidates = list(zip(...,
  strict=True))` will break.
- **`_widen_like_mo1` draws from the clause grids**, so grid values must **not**
  be in the MO-0 denylist (`regulated_literals` = *current values only*).
  Putting them in starves MO-0 into a silent DISPLAY fallback.
- **MO-0 must stay a matched numeric control** (CLAUDE.md #7). Match **total**
  edit counts across arms, not per-arm ranges — jittering both arms over the same
  range *shifts* rather than overlaps and regressed Gate E 0.757 → 0.914 once.
- **`PIC 9(n)` with n > 18 is invalid COBOL.**
- **Recurring defect pattern, name it when you see it:** *absence read as
  permission* — a missing comparator read as a ceiling; a missing prior read as
  `current*1.1`; a missing name in a PROCEDURE DIVISION read as "inert"; a
  `MutationRejected` read as a statistic; **a floor that counts rows read as
  permission to duplicate.** Every defect this task found is an instance.
