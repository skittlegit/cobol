## Objective
`static_analysis/slicer.py`: `slice_on(var, programs, call_graph, program=None) ->
tool_types.Slice` — the minimal cross-paragraph statement set affecting `var`: backward slice
over data dependence + control dependence. This is what turns scattered COBOL into one
readable "rule" for the agent.

## Deliverables — semantics (normative)
1. **Data dependence (transitive):** worklist starting at `{var}`; include every def site of
   each worklist variable; identifiers *used* by an included def statement join the worklist.
   Fixpoint over finite sets — no depth cap.
2. **Control dependence:** every included statement pulls in its guarding conditions —
   ancestor IF/EVALUATE/WHEN/PERFORM_LOOP(UNTIL) statements via the `Statement.children`
   nesting — and identifiers in those conditions join the worklist.
3. **Interprocedural glue:** when the slice spans paragraphs, include the PERFORM_CALL/GOTO
   statements that connect included paragraphs (call-graph path edges between them), so the
   slice reads as a coherent flow, not disconnected fragments.
4. **Declaration context:** DATA DIVISION `VALUE`-clause def sites of sliced variables are
   included as SliceStatements (original-source text via LineMap — these carry the hardcoded
   reference lists D4 drift targets).
5. **Ambiguity policy:** `?ambiguous` sites from T1.3 are included (conservative
   over-approximation), never silently dropped.
6. **Ordering & shape:** statements sorted (program, original line); `paragraphs` = every
   NodeRef contributing ≥1 statement (`<preamble>` included when driver code contributes);
   `is_interprocedural` = statements span >1 paragraph or >1 program.
7. `scoped_program` semantics identical to T1.3 (A2 amendment).

## Gate (write first) — `tests/test_slicer.py` ⛳ M1
Golden fixtures `tests/fixtures/slices/<name>.json` — exact ordered (program, line, text)
sets, **hand-built by eyeballing source once** — for **10 slices**. Composition requirements:
- ≥3 interprocedural (cross-paragraph; ≥1 crossing via a PERFORM chain of depth ≥2);
- ≥1 rooted at a copybook-declared variable (copybook-stem ref appears in the slice);
- ≥1 whose guards include a nested IF-inside-IF and ≥1 an EVALUATE/WHEN;
- ≥1 including `<preamble>` statements (CBTRN02C driver);
- ≥1 pulling a VALUE-clause declaration def (the D4 pattern);
- ≥1 deliberately trivial (single paragraph, 2–4 statements) proving the slicer doesn't
  over-include.
Reuse T1.3's traced variables where possible (`WS-MONTHLY-INT`, `WS-TOTAL-INT`,
`ACCT-CURR-BAL`, `DALYTRAN-AMT` are good roots). Asserts: exact match per fixture;
`is_interprocedural` flag correct per fixture; full suite stays green; skip-marked without
corpora.
