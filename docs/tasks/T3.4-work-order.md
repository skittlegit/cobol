# T3.4 Work Order — Tiered Finding Verification

**Owner:** Track C · **Type:** [code] · **Status:** todo

## Preconditions

T3.1–T3.3 retrieval components are available, and the frozen finding/tool
shapes in `docs/CONTRACT.md` remain unchanged.

## Objective

Implement `verify(finding)` with the contract's three evidence tiers:

1. executed behavior with GnuCOBOL when runnable;
2. static support from the parser, graph, dataflow, and slice; and
3. clause-to-code entailment when execution is unavailable.

Tier 3 must use a verifier from a different model family than the system under
test. Unsupported citations are rejected, and every result records its tier.

## Deliverables

- `src/cobol_archaeologist/model/verify.py`;
- verifier fixtures spanning supported, unsupported, and execution-unavailable
  findings; and
- tests for tier selection, citation validation, family separation, and
  deterministic offline fallbacks.

## Gates

1. Validate at least 50 labelled finding/evidence pairs.
2. No finding is marked verified without a valid clause citation and code
   locus.
3. Tier selection is explicit and reproducible; CICS compile failure routes to
   Tier 2/3 rather than becoming a tool error.
4. Same-family model verification is refused.

## Out of scope

Agent orchestration (T3.5), policy-hunt generation (T3.6), and benchmark
headline evaluation (T4.x).
