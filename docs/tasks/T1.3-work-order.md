## Deliverables

### D1. `trace_variable(var, programs, call_graph, program=None) -> tool_types.VariableTrace`
`program=None` → corpus-wide; set → that program only (contract amendment A2 semantics).
Classification must be **AST-based** (walk the tree-sitter tree; how you access it is your
choice — extending the parser with an internal hook is fine, but `Statement` stays as-is
unless impossible, in which case stop). Def/use semantics (normative table):

| Statement | use | def |
|---|---|---|
| `MOVE a TO b…` | a | every b |
| `COMPUTE x = expr` | identifiers in expr | x |
| `ADD/SUBTRACT/MULTIPLY/DIVIDE … GIVING g` | operands | g |
| same, no GIVING (`ADD a TO b`) | a, b | b (emit both sites for b) |
| `INITIALIZE x` / `ACCEPT x` | — | x |
| `SET x TO y` | y | x |
| `IF / EVALUATE / WHEN / PERFORM … UNTIL` conditions | every identifier | — |
| `READ f INTO x` / `STRING/UNSTRING … INTO x` | — | x |
| `WRITE rec FROM x` / `DISPLAY x` | x | — |
| DATA DIVISION `VALUE` clause | — | def, `statement_kind="VALUE-clause"` |

Every site's `ref` is original-source (LineMap for copybook-expanded fields). Parse with
`include_preamble=True` — defs/uses in driver code must be visible. `excerpt` = the single
statement's text, **read from original source via ref** (preprocessed text may contain
literal-splice placeholders).

### D2. Name resolution
- Qualified references `X OF/IN GROUP` resolve to the specific field; bare `X` unique in a
  program resolves plainly; bare `X` declared in multiple records of the same program →
  include all candidate sites with `statement_kind` suffixed `"?ambiguous"` (never guess
  silently).
- `REDEFINES`: when the traced field's storage is redefined by another field (or vice versa),
  append one site-independent marker: `VariableTrace.sites` gains no fake entries; instead
  document via `statement_kind="REDEFINES-alias"` def-site at the redefining declaration.
  Full alias analysis is explicitly out of scope — module docstring states the limitation.
- 88-level condition names in conditions count as **use of the parent field** (documented).
