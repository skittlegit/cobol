## Objective
Sandboxed compile + execute for GnuCOBOL-compilable COBOL, plus the compile oracle Track B's
mutation pipeline calls on every generated instance.

## Deliverables — `model/run_cobol.py`

### D1. `compile_check(source: str) -> CompileResult`
New small model (this module's own, NOT tool_types — it's a Track A/B internal, not an agent
tool): `CompileResult{ok: bool, messages: list[str]}`. Implementation: `cobc -fsyntax-only
-std=ibm` in a temp dir; messages = parsed error/warning lines (file:line: text). This is the
T2.2 oracle — it must be cheap (<1s typical) and safe to call hundreds of times.

### D2. `run_cobol(source: str, inputs: RunInputs | None = None) -> RunResult`
- Fresh temp dir per call; write `inputs.files` into it; compile `cobc -x -std=ibm`;
  on compile failure → `RunResult{compiled_ok=False, stderr=<cobc output>}` — **never raise**
  (CICS programs failing here is expected: "Tier-1 verification unavailable" semantics).
- Execute with `inputs.stdin`, **timeout 5s** (`timed_out=True`, exit_code=None on kill),
  cwd = the temp dir, subprocess env minimal (no inherited PATH surprises beyond cobc/libcob),
  no network access assumed or needed. Capture stdout/stderr (cap each at 64KB, note
  truncation in the string).
- File outputs the program writes into its cwd are the caller's business: add
  `RunResult`-adjacent helper `collect_outputs(tmpdir) -> dict[str, str]`? **No** — keep the
  frozen RunResult shape; instead `run_cobol(..., keep_dir=False)` internal param, and expose
  `run_cobol_with_files(...) -> tuple[RunResult, dict[str, str]]` as a module-level extra for
  T6.2's future test capture. RunResult itself stays contract-exact.
- `cobc` discovery: `COBC` env var override, else `shutil.which("cobc")`; missing →
  `RuntimeError` with the one-line install hint (this IS an error, unlike compile failure).

### D3. Environment bootstrap
`scripts/setup_cobc.sh`: idempotent `apt-get install -y gnucobol3` + `cobc --version` echo.
CI/test skip rule: harness tests are skip-marked when `cobc` is absent (same pattern as the
corpora skip) — the suite must stay green on machines without GnuCOBOL.

## Gate (write first) — `tests/test_run_cobol.py`
1. Trivial fixture program (DISPLAY literal) compiles and runs; stdout asserted.
2. A real CardDemo **batch** program slice end-to-end: take the `WS-MONTHLY-INT` slice
   region's host program `CBACT04C` — compile the full program with stub input files and
   assert it executes (exit code captured; CardDemo batch programs read VSAM-style files —
   provide minimal empty/1-record fixtures via `inputs.files`; if file-control semantics make
   a clean run impractical, a compile-only assertion on CBACT04C + full run on the trivial
   program satisfies the gate — say which in the commit).
3. Timeout: an infinite-loop fixture returns `timed_out=True` in ~5s, no zombie process
   (assert the pid is gone).
4. CICS negative case: `COSGN00C` → `compiled_ok=False`, stderr non-empty, no exception.
5. `compile_check` on a deliberately broken fixture returns `ok=False` with a parsed
   file:line message.
6. Skip-marked without `cobc`; suite green either way.
