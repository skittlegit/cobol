## Objective
Any CardDemo program → clean parse (zero ERROR nodes) → typed `Program` with exact paragraph
spans, statement nesting, and all coordinates in **original-source** line numbers.

## Deliverables

### 1. `ingest/cleaner.py` — preprocessor
`preprocess(source: str) -> PreprocessResult` with `text` (line count == input, assert it) and
`masked_spans: list[MaskedSpan{kind, start_line, end_line, original_text}]`,
`kind ∈ {exec_cics, exec_sql, exec_dli, copy_replacing}`. Rules (port from spike):
- `EXEC CICS|SQL|DLI … END-EXEC` → first line `CONTINUE`, interior blank; **emit `CONTINUE.`
  when the `END-EXEC` line ends with `.`** (load-bearing; see spike).
- Comment lines (col 7 `*` or `/`) never scanned for EXEC/COPY.
- Sequence area (cols 1–6) and cols >72 must not affect scanning.
- `masked_spans.original_text` must be verbatim — T1.2's LINK/XCTL extractor reads it.

### 2. `parser/paragraphs.py` — AST parse
`parse_program(path, backend="ast") -> Program` (pydantic):
`Program{program_id, path, paragraphs: list[Paragraph]}`;
`Paragraph{span: tool_types.ParagraphSpan, statements: list[Statement]}`;
`Statement{kind, ref: tool_types.SourceRef, children: list[Statement]}` where
`kind ∈ {IF, ELSE, EVALUATE, WHEN, PERFORM_CALL, PERFORM_LOOP, MOVE, COMPUTE, OTHER}`.
Grammar facts (from spike — trust these): no `paragraph` container node → compute spans as
paragraph_header → next header/section/division end; control flow appears as
`if_header/else_header/END_IF`, `evaluate_header/when/END_EVALUATE`; PERFORM targets are
`perform_procedure` under `perform_statement_call_proc`; loops are `perform_statement_loop`.
PERFORM_CALL statements carry `target: str` (and `thru_target: str | None`).
`backend="regex"` fallback: same `Program` shape, `statements=[]`.

### 3. `parser/copybooks.py` — real expansion + LineMap
`expand(source: str, search_paths: list[Path]) -> Expansion{text, line_map:
list[tool_types.LineMapEntry]}` resolving `COPY X` and `COPY X REPLACING ==a== BY ==b==`
(pseudo-text replacement). CardDemo search paths: `app/cpy/`, `app/cpy-bms/`. Copybook file
resolution is case-insensitive on name and extension (corpus mixes `.cpy`/`.CPY`; programs may
also have `.CBL`). Nested COPY: support one level; deeper → stop condition.
Pipeline order: expand → preprocess → parse; every public line number round-trips through the
LineMap back to (file, original line).

COTRN02C, COUSR02C` (note `CBSTM03A.CBL` uppercase extension — glob both cases). Fixture =
list of (name, start_line, end_line) built by regex extraction, hand-eyeballed once. Asserts:

2. **Zero ERROR nodes** after preprocessing, all 10.
3. Nesting spot-checks: one hand-verified IF-inside-IF and one EVALUATE (pick from CBTRN02C
   and COACTUPC), exact `Statement` tree asserted.
4. Line-count invariant: `preprocess(src).text` has identical line count for all 10.
5. A copybook-expanded field's declaration resolves via LineMap to the correct
   (copybook file, line) — one fixture case using `CVACT01Y` (account record) or equivalent.
6. Skip-marked when `data/corpora/` absent (CI without corpus stays green).

## Stop conditions

  rule → stop, report the pattern + the program; the chat adjudicates (never patch `vendor/`).
  Any new preprocessor rule gets logged in `docs/preprocessor-notes.md` + a regression fixture.
- COPY nesting deeper than one level in a gate program → stop, report.
- Any need to change `tool_types.py` → CONTRACT CHANGE, stop immediately.
