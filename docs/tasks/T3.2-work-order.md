## Objective

Build the retrieval stack for `search_regulations`: BM25 + dense embedding
retrieval fused with RRF, plus a cross-encoder reranker. Demonstrate on the
fixed query set that hybrid+rerank beats dense-only. This lands the retrieval
engine; the `ToolLayer.search_regulations` wiring itself is T3.5's seam.

## Files to create

1. `src/cobol_archaeologist/rag/index.py` — corpus build + BM25 + dense +
   RRF fusion + rerank. Public API:
   `RegulationIndex.build(chunks) / search(query, k=5, doc=None, mode=...)`
   where `mode ∈ {"bm25", "dense", "hybrid", "hybrid_rerank"}` and `doc`
   optionally restricts to one document (T6 will need old-side-only search).
2. `src/cobol_archaeologist/rag/embed.py` — embedder + reranker wrappers,
   models pinned below, with a local cache dir; no hub calls when cached.
3. `tests/test_retrieval.py` — gates below.
4. `tests/fixtures/retrieval/queries.jsonl` — committed as provided.
5. The generated relevance-evidence section in this work order.

## Pinned models (determinism + on-prem framing, T7.2)

## Corpus and gold resolution

- Index chunks from **all** documents the chunker emits (old-side 2016/2022
  docs included — they are realistic distractors and T6 needs them).
- Gold for each query resolves **live**: `record_id` → `(clause.doc,
  clause.clause_id)` via `data/regulations/clauses.jsonl` at test time. A hit
  = a returned chunk with matching (doc, clause_id). PROVISIONAL ids join
  as-is; when T2.5 renames them, the query set survives untouched.
