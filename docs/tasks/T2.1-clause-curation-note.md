## Version-encoding rule (binding for T2.3/T2.5)

Per clause, not per document: a clause amended on 2024-03-07 (per the consolidation's
amendment footnotes) carries `version: "2024-03-07"`, `effective_date: 2024-03-07`; an
unamended clause carries `version: "2022-04-21"`, `effective_date: 2022-07-01`. Exception

## CurrentValue conventions (the T2.1 vocabulary the schema comment delegates to Track B)

`CurrentValue` is frozen as `{kind, value}` with `extra="forbid"` ‚Äî confirmed by gate failure
when qualifiers were placed top-level. Conventions:

- **kinds:** `duration_days`, `duration_working_days`, `duration_months`, `duration_years`,
  `amount_inr`, `percentage`, `multiplier`, `enum_set`, `boolean_required`, `formula`,
  `composite`.
- **Qualifiers** (`comparator`: `at_least` / `at_most` / `strictly_greater`; `note`) live
  **inside** `value` as `{"amount": X, "comparator": "..."}` ‚Äî never as extra fields.
- **`composite`**: `value` is a dict of named sub-values, each itself `{kind, value}`-shaped.
  Consumers (T2.2 mutation targeting, T4.2 D1/D5 scoring) must handle both bare-scalar and
  qualified-dict `value` shapes.
- The `comparator` qualifier is what makes D5 objectively scorable: 9(b)(v)'s
  `strictly_greater` on 3 days is the canonical `>` vs `>=` ground truth.
