    "not_equal",         # <>

### `CurrentValue` — now recursive and typed

```python
Scalar = int | float | str | bool | list[str]

class CurrentValue(BaseModel):
    model_config = ConfigDict(extra="forbid")
    kind: str                                    # free vocabulary (B extends)
    value: Scalar | dict[str, "CurrentValue"]    # dict ⇔ kind == "composite"
    comparator: Comparator | None = None         # leaves only
    note: str | None = None
```

Validator (`mode="after"`):
- `kind == "composite"` ⇒ `value` **must** be a mapping, and `comparator`
  **must** be `None` (comparators belong to leaves, not to composites).
- `kind != "composite"` ⇒ `value` must **not** be a mapping.

Watch the union: with `bool` and `int` both admissible, confirm pydantic's smart
union does not coerce `True → 1`. Add a test.

Ship a module-level helper:

```python
def resolve_path(cv: CurrentValue, path: str) -> CurrentValue: ...
```
Dotted path into a composite (`"penalty_per_day"`, `"a.b"`). Raises `KeyError`
on a bad path. Both B's emitters and C's T4.2 metrics import this — it is the
single canonical accessor.

### `SourceLocus`, `SourceLineRef`, `CodeLocus`, `Labels` — flat fields replaced

```python
class SourceLocus(BaseModel):
    model_config = ConfigDict(extra="forbid")
    program: str
    paragraph: str | None = None      # None when not inside a paragraph (WS, decls)
    file: str | None = None           # None ⇒ the program's own source;
                                      # else the copybook/file the line resolves to
    line_span: tuple[int, int]        # 1-based; validator: both ≥1 and start ≤ end

class SourceLineRef(BaseModel):
    model_config = ConfigDict(extra="forbid")
    program: str
    line: int                         # ≥ 1
    file: str | None = None

class CodeLocus(BaseModel):
    model_config = ConfigDict(extra="forbid")
    loci: list[SourceLocus] = Field(min_length=1)
    slice_vars: list[str]
    is_interprocedural: bool          # required, no default (stratification key)

class Labels(BaseModel):
    model_config = ConfigDict(extra="forbid")
    program_level: Literal["drift", "conformant"]
    paragraph_level: Literal["drift", "conformant"]
    line_level: list[SourceLineRef]   # [] for conformant
```

`CodeLocus` validator: if `len({l.program for l in loci}) > 1` then
`is_interprocedural` must be `True`. **One-way only** — `is_interprocedural`
also covers cross-paragraph single-program cases (playbook §1E), so a single-
program locus may legitimately be either.

### `DriftInstance`

```python
class DriftInstance(BaseModel):
    model_config = ConfigDict(extra="forbid")
    instance_id: str = Field(pattern=r"^drift_\d{6}$")
    regulation_clause: RegulationClause
    code_locus: CodeLocus
    drift_type: DriftType
    target_path: str | None = None    # dotted path into current_value; NEW
    labels: Labels
    gold_rationale: str = Field(min_length=1)
    provenance: Provenance
```

Validators (`mode="after"`) — the three v1 rules **preserved verbatim**, three new:

1. *(v1)* `D7_conformant` ⇒ `program_level == paragraph_level == "conformant"`
   and `line_level == []`.
2. *(v1)* non-`D7` ⇒ `program_level == "drift"`.
3. *(v1)* `source == "synthetic"` and non-`D7` ⇒ `provenance.mutation` is not None.
4. **(new) line/locus integrity:** every `SourceLineRef` in `line_level` must
   match some `SourceLocus` on `(program, file)`, with
   `locus.line_span[0] <= ref.line <= locus.line_span[1]`.
5. **(new) target_path required for composite D1/D5:** if
   `regulation_clause.current_value` is not None and its `kind == "composite"`
   and `drift_type in {"D1_stale_threshold", "D5_boundary_error"}`, then
   `target_path` must be present, must `resolve_path` without error, and must
   land on a **non-composite** node.
6. **(new)** `target_path` present ⇒ `current_value` is not None and the path
   resolves.

## F8 fix (`rag/index.py`) — independent, folded in

`search(..., mode="hybrid")` returns `Hit(chunk, 0.0)` at `index.py:226`:
every hybrid hit carries score **zero**, because `reciprocal_rank_fusion`
returns a ranked id list but never its RRF scores. Ranking is unaffected (the
T3.2 relevance table stands, no re-run needed), but T3.5's agent will read
`.score` and get garbage.

Fix: `reciprocal_rank_fusion` returns `list[tuple[int, float]]` (candidate
index, RRF score); `hybrid` mode passes the score into `Hit`. `hybrid_rerank`
keeps the cross-encoder score, unchanged. Add a test asserting **no mode**
returns an all-zero score vector for a query with hits.
