# Work Order — T4.3: Trajectory and evidence-path evaluation

**Owner:** Track C · **Type:** [code] · **Effort:** M · **Status:** ratified
**Precedence:** this work order > `CLAUDE.md`.

## Objective

Audit whether each answer followed an admissible, replayable evidence path
rather than merely matching a label.

## Deliverables

- `src/cobol_archaeologist/eval/trajectory.py`
- `tests/test_trajectory_eval.py`

## Checks

- Tool requests, bounded observations, model responses, and the full
  `VerificationResult.tier_attempts` ladder are present.
- A finding follows at least one class-required evidence path from T3.6.
- The cited code fact appears in a successful tool observation and overlaps a
  gold typed locus; string occurrence alone is insufficient for D6.
- No filesystem, git, mtime, mutation provenance, or gold-label shortcut
  appears in tool arguments or observations.
- Budget counts recompute from the trajectory and agree with recorded totals.
- Failed or unavailable Tier-1 attempts remain distinguishable.

## Output

One `TrajectoryAssessment` per evaluation record:
`replayable`, `evidence_path_ok`, `code_fact_ok`, `budget_ok`,
`shortcut_free`, and typed reasons.

## Gates

Golden good, under-evidenced, truncated, budget-mismatched, shortcut-bearing,
and Tier-1-unavailable trajectories. Assessment is deterministic and never
upgrades an unverified finding.
