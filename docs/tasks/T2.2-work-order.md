## What this builds

Operators that turn a **conformant** program + a clause record
(`data/regulations/clauses.jsonl`) into a **labeled drifted copy** ‚ÄĒ plus the
benign-edit operator that keeps conformant instances indistinguishable from
edited ones. Every emission is a valid `DriftInstance` (frozen schema; any
change needed = STOP, CONTRACT CHANGE).

| Operator | Drift | Transformation | Targeting source |
|---|---|---|---|
| MO-0 | D7 | benign edits on conformant code (see catalogue) | any program |
| MO-1 | D1 | replace regulated literal with an outdated/wrong value | record's `current_value` (+ `prior_2022`/`prior_versions` values as realistic stale sources) |
| MO-2 | D2 | delete the IF branch / PERFORM implementing a required check | `candidate_locus` paragraph |
| MO-3 | D3 | invert comparator / remove blocking branch so forbidden cases pass | condition in the check slice |
| MO-4 | D4 | remove/alter an entry in a hardcoded 88-level list | copybook lists (CSLKPCDY idiom) |
| MO-5 | D5 | swap `>`‚ÜĒ`>=` / `<`‚ÜĒ`<=` or shift boundary by 1 | comparator per record's `comparator` qualifier |
| MO-6 | D6 | wrap the compliance check in an always-false flag guard | `WS-` flag idiom per T0.2 ¬ßD6 |
| MO-1√ó | D1 | stale value lives in a **copybook constant** far from use | CC-10h cutoff constants (designated showcase) |
| MO-3√ó | D3 | blocker intact in one paragraph; gate weakened/second path added **elsewhere** | CBTRN02C validate‚Üígate‚Üípost shape (T0.2 ¬ßD3) |
| MO-6√ó | D6 | disabling flag set in a **different program** in the batch chain | CBTRN01C‚ÜíCBTRN02C sequence |

**Targeting map is data, not code:** iterate `clauses.jsonl`; each record's
`check.mutation_ops` lists its legal operators and `check.candidate_locus`
its host. Records with `candidate_locus.kind == "synthetic"` are skipped until
a matching seed program exists (report the skip count).

## Design requirements (binding)

1. **API:** `mutate(base: ProgramSource, record: ClauseRecord, op: str,
   rng: Random) -> MutationResult` where `MutationResult` carries the mutated
   source, a full `DriftInstance`, and a `validation` block (below).
   Deterministic under a fixed seed ‚ÄĒ same inputs, same mutation.
2. **Line-number fidelity is sacred:** all `labels.line_level` and
   `code_locus.line_span` refer to the **mutated file's original-format
   lines** via LineMap. D2 deletions label the **insertion point** (CONTRACT
   Part 3 convention). `code_locus.slice_vars` comes from `slice_on` over the
   touched variable(s); `is_interprocedural` is set by the operator variant
   (base = false, √ó = true) and must agree with the slice's program/paragraph
   spread ‚ÄĒ assert this.
3. **Validation ladder, recorded per instance** in
   `provenance.mutation` (append `; validation=<level>`):
   - `compiled` ‚ÄĒ GnuCOBOL-native base: `run_cobol` compile (and run when the
     harness fixture supports it). Required level for seed-program bases.
   - `syntax` ‚ÄĒ CICS bases: `cobc -fsyntax-only` on the **preprocessed/masked**
     form (raw CICS won't parse; the cleaner's masking is the pipeline norm).
   - `ast` ‚ÄĒ fallback only if cobc is unavailable: reparse mutated source,
     assert zero ERROR nodes and paragraph structure unchanged outside the
     touched region. If used, say so loudly in the summary.
   A mutation failing its ladder level is rejected, not downgraded.
4. **MO-0 benign-edit catalogue** (`surface.py`): rename non-regulated
   identifiers (preserve PIC/usage); reword/reflow comments; reorder
   independent paragraphs (call-graph-safe only ‚ÄĒ use `call_graph.py` to
   verify no ordering dependency); perturb **non-regulated** literals
   (display strings, cosmetic counters ‚ÄĒ never anything a record's
   `current_value` maps to). Every conformant instance in any emitted set
   gets ‚Č•1 MO-0 edit. MO-0 must also run on **drifted** outputs (style
   diversification pass) so operator fingerprints wash out.
5. **Style diversification:** implemented as a pluggable pass
   (`diversify(source, region) -> source`). For this task's gate, the
   deterministic surface-transformer suffices (synonym tables for comments,
   identifier renamer, reflow). The LLM-paraphrase backend is wired behind
   the same interface but exercised at T2.3 scale, not here. Re-run the
   validation ladder after diversification ‚ÄĒ a diversified mutant that no
   longer validates is rejected.
6. **Plausibility floor:** mutated values must look like real drift ‚ÄĒ MO-1
   draws stale values from the record's actual prior versions when present
   (e.g. 15 for the BO threshold, "per day" semantics for ¬∂19) before falling
   back to plausible-wrong generation (round numbers near the true value;
   never absurd magnitudes). Absurdity screening proper is T2.4's job.

## Gate tests (`tests/test_mutate.py`) ‚ÄĒ write FIRST

- **Gate A (coverage):** every operator including MO-0 and all three √ó
  variants produces ‚Č•1 validated instance against the anchor loci named in
  the targeting map; √ó variants have `is_interprocedural=True` and multi-file
  or multi-paragraph `code_locus`.
- **Gate B (contract):** every emitted `DriftInstance` round-trips the frozen
  schema; D2 line labels non-empty (insertion point); `provenance.source ==
  "synthetic"`, `base_program` set, `mutation` string names operator + locus
  + old/new values; determinism: two runs with the same seed byte-identical.
- **Gate C (anti-gaming probe):** generate a balanced probe set (‚Č•50 drifted,
  ‚Č•50 MO-0-edited conformant, mixed bases). A logistic-regression probe over
  surface-only features ‚ÄĒ diff size, touched-line count, comment-density
  delta, identifier-entropy delta, literal "roundness", whitespace churn ‚ÄĒ
  must score **AUC whose 95% bootstrap CI contains 0.5**. If it doesn't, the
  operators are leaking edit signal: fix surface.py, don't loosen the gate.
- **Gate D (ladder honesty):** every instance's validation level recorded;
  zero `ast`-level instances when cobc is present.
