## Per-class detection contract

Each hunt declares the evidence it must obtain before emitting. Missing
evidence → abstain, never a guess.

| Class | Primary evidence | Required fields |
|---|---|---|
| **D1** stale threshold | literal at locus vs `current_value` (via `resolve_path` when composite) | `target_path` **required** for composite clauses |
| **D2** missing rule | absence across the searched scope — `grep` + `find_callers/callees` + slice, all negative | `line_level` = **insertion point(s)**, per CONTRACT.md Part 3 (B's sign-off note 2) |
| **D3** contradictory | two loci with conflicting outcomes for the same regulated condition | ≥2 `loci`; `is_interprocedural=True` when they span programs |
| **D4** stale reference data | hardcoded enumeration vs clause's `enum_set` | the missing/extra entries named in `gold_rationale` |
| **D5** boundary/comparator | comparator at locus vs clause `comparator` (typed, any depth) | `target_path` required for composite |
| **D6** dead compliance code | reachability — see below | reachability evidence in the trajectory |
| **D7** conformant | positive confirmation the check exists **and** matches | `line_level=[]`, both label levels `conformant` |

**D6 semantics (Track A flag, 2026-07-17) — mandatory.** Consume
`forest_roots` + `reachable_from`; `entry_points` is the reachability **seed
only**, never the deadness oracle. Reachability traverses
`edge_kind="fallthrough"`; a fall-through-reached paragraph has no caller yet
is live, so caller-absence is **not** evidence of deadness. `verify.py::
_tier2_reachability` already implements this — **delegate to it, do not
reimplement.**

**D7 is not the default.** A conformant verdict requires positive evidence that
the check exists and matches the clause. "Found nothing" is D2 or abstention,
never D7. Assert this: a hunt over a scope with no relevant code must not
return D7.

## Anti-shortcut rule (integrity — non-negotiable)

Track B applied MO-0 benign edits and style diversification precisely so that
"was this file edited" carries **zero** label information (their probe:
AUC 0.50 on six surface features). Hunts must classify from **semantics vs the
clause** — never from edit-artifact cues: comment freshness, formatting
discontinuity, identifier-naming style, or literal "roundness". No hunt may
read git history or file mtimes. State this in `policy.py`'s docstring; it is
the property the benchmark's validity rests on.

## Emission rules

- **Verification before emission** — build a `Finding`, call `verify()`,
  abstain on failure. Inherited from T3.5; assert it holds per-hunt too.
- **Typed loci only** — every `SourceLineRef` binds to a program (and `file`
  for copybook loci). No flat or inferred line ownership. Multi-program loci
  force `is_interprocedural=True` (schema-enforced).
- **Confidence + tier retained** on every finding; trajectory provenance
  attached so T4.3 can score the path.
- **Abstention is a first-class outcome.**

## Fixtures (`tests/fixtures/hunts/`)

one positive per class D1–D7; one **interprocedural** (D3× or D6×, spanning two
programs); one **copybook** case (MO-1× — threshold in a copybook, `file` set);
one **temporal pair** from the 5 T6 pairs (same code, two clause versions,
opposite verdicts); one **conformant** (D7, MO-0 edited — proves the hunt isn't
reading edit artifacts); one **abstention** (insufficient evidence).

4. Verification tier + trajectory provenance retained on every emitted finding.
5. **D6 delegates to `verify.py::_tier2_reachability`**; a test asserts

6. **D7 requires positive evidence** — empty scope yields abstention or D2, not D7.
7. **Composite-clause D1/D5 carry a resolving `target_path`** (schema-enforced;
