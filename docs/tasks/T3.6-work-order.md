# T3.6 Work Order — Drift Policy Hunts

**Owner:** Track C · **Type:** [code] · **Status:** todo

## Preconditions

T3.5 provides a bounded investigation loop and T3.4 provides tiered
verification. Benchmark inputs conform to the frozen `DriftInstance` schema.

## Objective

Implement policy hunts that localize and classify D1–D7 drift. A hunt returns
a `DriftInstance`-shaped finding with clause citation, typed code loci,
confidence, and verification tier, or explicitly abstains.

## Deliverables

- policy-hunt orchestration under `src/cobol_archaeologist/model/` or
  `src/cobol_archaeologist/agent/` as assigned by ownership;
- one registered hunt for every D1–D7 class; and
- fixtures covering local, interprocedural, copybook, temporal, conformant,
  and abstention cases.

## Gates

1. Every D1–D7 hunt produces a schema-valid result on a positive fixture.
2. Interprocedural findings bind each line to its program/file through typed
   loci; no flat or inferred line ownership is accepted.
3. Unsupported or insufficient evidence produces abstention, not a fabricated
   drift label.
4. Verification tier and trajectory provenance are retained on every emitted
   finding.

## Out of scope

Headline metric computation (T4.x), annotation (T5.1), and migration (T7.x).
