# T3.5 Work Order — ReAct Investigation Loop and Tool Seam

**Owner:** Track C · **Type:** [code] · **Status:** todo

## Preconditions

The frozen ToolLayer protocol, retrieval stack, and T3.4 verifier are present.
Development may use `StubToolLayer`; the real seam must remain a constructor
swap and must honor the semantics embedded in `T1.6-work-order.md`.

## Objective

Build the bounded ReAct loop that investigates a policy question, calls only
the injected ToolLayer, records a replayable trajectory, emits typed findings,
and abstains when evidence is insufficient.

## Deliverables

- `src/cobol_archaeologist/agent/` loop and stub tool layer;
- prompt/trajectory models compatible with `docs/CONTRACT.md`;
- explicit tool-call and token/step budgets; and
- a smoke trajectory for a late-fee policy query.

## Gates

1. A deterministic stub run completes the late-fee investigation and produces
   a schema-valid finding or explicit abstention.
2. The loop cannot exceed the configured tool-call/step budget.
3. Every tool request, observation, citation, verifier result, and final answer
   is logged for replay.
4. Replacing the stub with `RealToolLayer` requires no agent-loop code change;
   sentinel, truncation, lookup-error, and Tier-1-unavailable semantics match
   T1.6.

## Out of scope

Policy-hunt benchmark coverage (T3.6), MCP wrapping (T7.1), and performance
tuning.
