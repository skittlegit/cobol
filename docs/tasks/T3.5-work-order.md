## Objective

A bounded ReAct loop that investigates a policy question by calling **only** the
injected `ToolLayer`, records a replayable trajectory, emits a verified
`DriftInstance`-shaped finding, and **abstains** when no tier verifies.

## Part 1 — `StubToolLayer` (build first, it is the test substrate)

Deterministic, no network, no models. Backed by a small committed fixture
corpus under `tests/fixtures/agent/corpus/` (2–3 small COBOL programs with a
late-fee rule, plus a handful of clause records).

**Contractual requirement (CONTRACT.md Part 1, stub parity clause):**
`isinstance(StubToolLayer(), ToolLayer)` must hold, and every method must
return the **same pydantic models** from `tool_types.py` that `RealToolLayer`
returns. Assert both in `tests/test_stub_tools.py`.

**Semantic parity, not just structural** — this is where the seam actually
breaks. Match `RealToolLayer` on: sentinel values for "not found", truncation
behavior at the ~60-line cap, lookup-error semantics, and
`run_cobol` Tier-1-unavailable behavior. The T1.6 register is normative; where
the stub cannot match, document the divergence in the module docstring rather
than silently differing.

## Part 2 — Trajectory (build second; the loop writes into it)

```python
class ToolCall(BaseModel):
    step: int
    tool: str
    arguments: dict[str, Any]
    observation_summary: str       # bounded; never a full dump
    observation_truncated: bool
    error: str | None = None
    latency_ms: float | None = None

class Trajectory(BaseModel):
    question: str
    steps: list[ToolCall]
    verification: VerificationResult | None   # from T3.4, verbatim
    finding: DriftInstance | None             # None iff abstained
    abstained: bool
    abstention_reason: str | None
    budget: BudgetSpec
    budget_exhausted: bool
    model_id: str
    seed: int | None
```

Trajectory must be **JSON round-trippable** and sufficient to replay the run
without re-calling the model. T4.3 (trajectory eval) and T4.4 (calibration)
both consume this; design it for them now rather than retrofitting.

`verification` holds the T3.4 `VerificationResult` **including its full
`tier_attempts` ladder** — do not flatten it to a tier number. The per-tier T4
faithfulness metric depends on knowing Tier 1 was attempted-and-unavailable
versus never attempted.

## Part 3 — The loop

Think → call one tool → observe → repeat, until a verified finding or
abstention. Requirements:

- **Budgets, enforced not advisory:** `max_steps`, `max_tool_calls`,
  `max_tokens`, wall-clock timeout. Exceeding any → terminate and **abstain**
  with `budget_exhausted=True`. A budget breach is never a finding.
- **Tool access only through the injected `ToolLayer`.** The loop must not
  import `RealToolLayer`, touch the filesystem, or call `rag/` directly.
  Constructor signature: `InvestigationLoop(tools: ToolLayer, ...)`.
- **Verification is mandatory before emitting.** Build a `Finding`
  (`Finding.from_prediction`, populating `exec_probe`/`static_claim` when the
  investigation produced them) and call `verify()`. `verified=False` →
  abstain. **The loop may not emit an unverified finding under any path**;
  assert this.
- **Abstention is a first-class outcome**, not an error. CONTRACT.md Part 3
  scores abstention explicitly; T4.4 needs the rate.
- **Determinism:** pin model + temperature 0 + seed as constants; cache model
  responses for the golden trajectory to `tests/fixtures/agent/` so the suite
  runs offline, same pattern as the T3.4 NLI cache.

## Gates

1. **Late-fee investigation, stub, deterministic** — completes and produces
   either a schema-valid `DriftInstance` or an explicit abstention. Golden
   trajectory committed.
2. **Budget enforcement** — a deliberately under-budgeted run terminates,
   abstains, sets `budget_exhausted=True`, and emits no finding.
3. **Replayability** — trajectory JSON round-trips; every tool request,
   observation, citation, verifier result, and final answer is present.
4. **Seam parity** — swapping `StubToolLayer` → `RealToolLayer` requires **no
   agent-loop code change** (constructor argument only). Assert via a test that
   constructs the loop with both and checks the loop module never imports
   `tools`. Sentinel / truncation / lookup-error / Tier-1-unavailable semantics
   match T1.6.
5. **No unverified emission** — a finding whose verification fails every tier
   yields `finding=None, abstained=True`.
6. **`isinstance(StubToolLayer(), ToolLayer)`** and return-model parity with
   `tool_types.py`.

Gates 1–3, 5, 6 must run **offline** (cached model responses). Gate 4's
`RealToolLayer` half may be `@pytest.mark.network`/corpus-gated.
