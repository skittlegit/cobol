# Work Order — T4.5: M4 baselines and go/no-go decision

**Owner:** Track C · **Type:** [code/report] · **Effort:** L · **Status:** ratified
**Precedence:** this work order > `CLAUDE.md`; CONTRACT Part 4 is normative.

## Objective

Compare the agent with dense-RAG and oracle-slice single-shot baselines on the
same frozen rows, then issue the Week-10 M4 go/no-go result without changing
the bars after seeing the data.

## Deliverables

- `src/cobol_archaeologist/eval/baselines.py`
- `src/cobol_archaeologist/eval/report.py`
- `tests/test_m4_report.py`
- `data/eval/m4/report.md` plus machine-readable `report.json`

## Baselines

1. Dense-RAG single-shot: clause retrieval plus bounded code context, no loop.
2. Oracle-slice single-shot: gold slice variables select Track A's real slicer;
   the model receives that slice but no labels/rationale.
3. Agent: T4.1 bounded policy loop.

All systems use the same model family, decoding settings, instance order, and
paired bootstrap samples. Provider failures remain paired unavailable rows.

## Frozen decisions

- T1 F1 must be at least 0.70 overall.
- Agent minus dense-RAG on the interprocedural stratum must be at least +0.10,
  with bootstrap 95% CI excluding zero and paired-test p < 0.05.
- Oracle-slice deconfounder is mandatory: if the agent does not beat it on the
  interprocedural stratum, reframe the contribution as the slicer before scale.
- T6 reporting bar is paired accuracy at least 0.70 on at least 20 pairs, with
  exact binomial interval. Insufficient pair count is `NOT EVALUABLE`, not pass.

## Gates

The report refuses a go/no-go verdict when artifacts, paired rows, provenance,
human verifier labels, or required baselines are missing. A valid no-go is a
completed checkpoint; an unevaluable run is not.

## Completion

M4 closes only when the report contains first numbers for all required systems,
confidence intervals, paired significance, tier faithfulness, coverage, T6,
and the oracle-slice decision.
