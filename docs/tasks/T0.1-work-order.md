# T0.1 Work Order — Corpus and Regulation Manifest

**Owner:** Track B · **Type:** [chat/data] · **Status:** done

## Objective

Create the canonical manifest for corpus roles, source pins, regulation
identity, versions, effective dates, and licences. Every later code or document
that names a corpus commit or regulation version must agree with this file.

## Deliverable

`data/manifest.json`, covering:

- AWS CardDemo as the Apache-2.0 anchor corpus, pinned at `59cc6c2fd7eb`;
- IBM CICS CBSA as the EPL-2.0 secondary corpus;
- the in-force 2025 RBI Commercial Banks Credit/Debit Card Directions and the
  KYC lineage incorporated by reference; and
- repo-native runnable benchmark bases, explicitly marked as not requiring an
  external commit pin and linked to `data/benchmark/seed/base_roster.json`.

## Gates

1. The manifest is valid JSON and all required source/version/licence fields
   are present.
2. Corpus-fetch tooling verifies a checkout against its recorded pin.
3. Regulation identifiers agree with `data/regulations/clauses.jsonl` and the
   source manifest.
4. Changes to anchor identity are reflected in this work order, CLAUDE.md, and
   downstream task work orders in the same reviewed change.

## Completion evidence

The manifest is checked in at v1.1. The 2025 re-anchor and repo-native base
provenance are recorded, and the corresponding source/roster tests are green.

## Out of scope

Clause-level curation (T2.1), source-PDF archival (T2.5), and corpus parsing
(T1.1).
