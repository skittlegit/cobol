## Objective

cross-program CALL + CICS LINK/XCTL edges, with `callers`/`callees`/reachability queries —
the backing for `find_callers`/`find_callees` (contract) and D6 dead-code detection later.

### D2. `static_analysis/call_graph.py`
```
build_call_graph(programs: list[Program],
                 preprocess_results: dict[str, PreprocessResult]) -> CallGraph
```
Edge sources:
1. `PERFORM_CALL` → edge to target; `thru_target` set → edges to every paragraph from target
   through thru_target **in Program.paragraphs order** (THRU is positional).
2. `GOTO` → edge (flagged `edge_kind="goto"` vs `"perform"` — reachability treats both as
   flow; later consumers may care about the difference).
3. `CALL 'LIT'` → cross-program edge to that program's first paragraph, `edge_kind="call"`.
   Dynamic CALL → record in `CallGraph.unresolved: list[UnresolvedCall{ref, reason}]` — never
   silently dropped.
4. **EXEC side-channel:** regex over `masked_spans.original_text` for
   `EXEC CICS (LINK|XCTL) PROGRAM('X'|WS-VAR)`; literal → cross-program edge
   (`edge_kind="link"|"xctl"`), identifier → `unresolved`. The masked span's start_line is the
   edge's `ref` line.
Node identity: `tool_types.NodeRef`. Unknown targets (PERFORM to a paragraph that doesn't
exist — possible in regex-fallback parses) → `unresolved`, not a crash.
API: `callers(ref) -> list[NodeRef]`, `callees(ref) -> list[NodeRef]`,
`reachable_from(entries: list[NodeRef]) -> set[NodeRef]`, and
`entry_points(program) -> list[NodeRef]` (first paragraph + any paragraph with no incoming
perform/goto edge from within the program).
