## Consumer-semantics register

This section is normative for ToolLayer meanings; `docs/CONTRACT.md` and
`tool_types.py` remain normative for shapes. `StubToolLayer` must reproduce
these semantics before the real/stub seam test.

### Sentinels and name conventions

- `<preamble>` is the synthetic leading paragraph for driver statements before
  the first real paragraph header. It is a first-class node and must not be
  filtered from reads, graph queries, traces, or slices.
- `NodeRef.paragraph == ""` means an external program entry whose first
  paragraph is unknown because the target was not parsed. The edge remains
  valid and queryable.
- `SourceRef.program` may be a copybook stem for a declaration site. Consumers
  must use `resolve_copybook` rather than assume every ref can be passed to
  `read_program`.
- A `statement_kind` suffix of `?ambiguous` means a bare field name resolved to
  multiple candidates. All candidates are returned conservatively; qualify the
  field with `OF`/`IN` to disambiguate. Matching is case-insensitive.
- `statement_kind == "REDEFINES-alias"` marks a redefining declaration that
  overlaps storage; it does not assert that a write through the alias was
  observed.
- An 88-level condition name traces to its parent field. In a data layout it is
  a level-88 child of that field with no PIC.

### Caps, truncation, and pointers

`ParagraphView.code` and `CopybookExpansion.text` are capped at 60 lines;
`GrepResult.matches` is capped at 200. When `truncated=True`, the accompanying
reference still spans the complete object and is the authoritative refetch
pointer. `ProgramView` deliberately returns paragraph names and spans, not
source dumps.

### Per-tool behavior

- `read_paragraph`, `grep`, declaration text, and VALUE evidence come from
  original source, never the preprocessed buffer.
- `find_callers` and `find_callees` expose invocation/transfer edges, not
  sequential fall-through. Unknown programs raise `ToolLookupError`; an
  unknown paragraph returns an empty list like a leaf.
- `entry_points(program)` is the single true entry. `forest_roots(program)` is
  the set of no-incoming-edge paragraphs excluding that entry. Internal
  `reachable_from` follows fall-through edges; D6 combines it with
  `forest_roots`.
- `trace_variable` and `slice_on` are corpus-wide when `program=None` and
  scoped when a program is named. An unknown scope raises.
- `resolve_copybook` returns LineMap entries naming real files; facade code
  replaces the expander's empty caller-source marker with the known path.
- `get_data_layout` returns integer levels and picture strings without the
  `PIC` keyword. Its source spans the complete original declaration. Continued
  VALUE literals must be read through that source/LineMap because T1.1 may use
  a placeholder in the parse buffer.
- `grep` is case-insensitive across original program and copybook sources. A
  copybook hit uses the copybook stem as `program`.
- `run_cobol` passes complete programs unchanged and wraps bare procedure
  snippets in a minimal shell with no invented storage. `compiled_ok=False`
  on CICS is evidence that Tier 1 is unavailable, not an exception. A missing
  `cobc` binary is an environment error and raises `RuntimeError`.
- `search_regulations` delegates to Track C's clause-anchored
  `RegulationSearch` implementation and returns `list[RegSearchHit]`, capped at
  five by the facade call. The T1.6 gate injects an offline search service so
  this seam is tested without model downloads.

### Known limitations

- LINKAGE/COMMAREA value flow is not threaded across program boundaries;
  corpus-wide name traces are a union, not parameter flow.
- Subscripts and reference modifications are normalized to the base field;
  their index/length identifiers are not separate sites.
- STRING/UNSTRING source operands are not modeled as uses; INTO targets are
  definitions per the locked def/use table.
- Computed `GO TO ... DEPENDING ON`, CICS transaction-table transfers, and
  COBOL `ENTRY` alternate entry points are not modeled. CardDemo contains none
  of the first/last forms; encounter one and stop rather than guess.
- Copybook expansion supports one nested level only.

### Errors

`ToolLookupError` (a `KeyError`) represents an unknown program, copybook,
record, or other required name. Return models describe valid answers and do not
use hollow objects as error variants.
