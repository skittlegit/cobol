## Phase 2 — Expand T6 to at least 20 real pairs

Add at least 15 new, independently authored verdict-flipping loci (at least 30
rows) while preserving the five existing valid pairs.

Each pair must:

- contain exactly two rows;
- have byte-identical `code_locus` objects and byte-identical underlying code;
- differ in regulation `version` and/or `effective_date`;
- have opposite correct program-level verdicts;
- use `provenance.source = "real_curated"` and
  `provenance.mutation = null`;
- cite archived, pinned primary evidence for both temporal sides;
- carry a rationale that explains why the same code changes verdict;
- use a distinct semantic code locus, not a copied pair with renamed
  identifiers; and
- compile with GnuCOBOL 3.2.0 when hosted in a seed program.

Anti-padding requirements:

- cover at least three independent regulatory lineages;
- add no more than five new pairs from one clause lineage;
- reject duplicate normalized program bodies and duplicate structural loci;
- do not count citation-only or same-verdict probes toward the 20-pair bar.

The frozen schema has no `pair_id`. Pair discovery remains structural equality
of `code_locus`; do not change `schemas.py` or `docs/CONTRACT.md`.

## Gates

All are hard gates:

1. Every seed and split row round-trips the frozen `DriftInstance` schema.
2. At least 20 structural T6 groups exist; every group has exactly two rows,
   byte-identical loci, two temporal keys, and opposite verdicts.
3. Every regenerated test row materializes successfully; the expected result
   is `N/N`, with no unavailable rows and no guessed replacements.
4. Every seed program compiles under GnuCOBOL 3.2.0.
5. All original T2.5 and T2.6 gates pass, including zero base leakage, real
   seed test-only, atomic pairs, class/stratum minima, and deterministic
   regeneration.
6. Newly authored T6 programs have no duplicate normalized body or duplicate
   structural locus.
7. All changed/new rows have current plausibility evidence; no stale judge
   result is attached to changed content.
8. Full offline repository tests and Ruff pass.
