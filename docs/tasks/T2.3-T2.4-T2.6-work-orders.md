## T2.3 — `benchmark-build` CLI + synthetic v1 (≥200 instances)

**Branch** `track-b/T2.3` · **Files** `src/cobol_archaeologist/cli.py`
(new subcommand), `benchmark/build.py` (orchestrator) ·
**Output** `data/benchmark/drift_instances.jsonl`

1. **CLI:** `cobol-archaeologist benchmark-build --seed <int> --out <path>
   [--min-instances 200] [--diversify deterministic|llm]`. Deterministic:
   same seed → byte-identical output file. The run manifest (operator counts,
   base counts, rejects by reason, validation-level counts, seed, git SHA)
   is written next to the output as `drift_instances.manifest.json`.
2. **Generation plan:** iterate the targeting map × legal operators × bases;
   apply MO-0 to every conformant emission; style-diversify every mutated
   region (T2.2 pass). Aim per-class floors, not exact balance (T2.6 balances):
   ≥20 per D1–D6, ≥60 D7, and **≥30 interprocedural** across MO-1×/3×/6× —
   the headline stratum floor. If a floor is unreachable with current bases,
   emit what exists and report the shortfall per class in the manifest —
   do NOT pad with implausible mutations.
3. **Diversification honesty:** `--diversify llm` requires an API key in env;
   if absent, refuse that mode (exit non-zero with a clear message) rather
   than silently falling back. Default mode is `deterministic`. Record the
   mode in the manifest and in every instance's `provenance.mutation` suffix.
4. **Validation ladder** (T2.2 rules) applies to every emission; rejects are
   counted, never silently dropped.
5. **Gate** `tests/test_benchmark_build.py`: (a) two runs, same seed →
   identical files; (b) ≥200 instances, all schema-v2 round-trip; (c) every
   class D1–D7 present; (d) interprocedural count ≥ 30 or manifest documents
   the shortfall; (e) probe classifier (T2.2 Gate C features) re-run on a
   200-instance sample: AUC CI contains 0.5.
6. STATUS: `T2.3 | done | B | data/benchmark/drift_instances.jsonl (N inst,
   manifest, gates green)`.

## T2.4 — Plausibility validation (LLM-judge + human spot-check)

**Branch** `track-b/T2.4` · **Files** `benchmark/judge.py`, `docs/tasks/
T2.4-judging-rubric.md` · **Two executors:** harness is [CODE]; the judging
run needs an external-family model and is [USER/CHAT].

1. **[CODE] Harness:** `judge.py` renders each instance into a judging prompt
   (clause text + mutated region ± 10 lines + the question "does this look
   like drift that could occur in real legacy code, or like an artificial
   edit?"), calls a configurable OpenAI-compatible endpoint, parses a
   {plausible|implausible|unsure, reason} verdict, writes
   `data/benchmark/judgements.jsonl`. **Hard rule encoded in the harness:**
   refuse to run if the configured model family matches the
   system-under-test family (Claude/Anthropic) — the integrity rule is
   different-family judging, and it is checked, not trusted.
2. **[CODE] Rubric doc:** what "plausible" means (looks like drift a real
   maintainer could leave behind; no style discontinuity; values not absurd),
   with 3 worked examples from the seed corpus.
3. **[USER] Judging run:** provide an OpenAI/Gemini key; run
   `benchmark-judge --sample 50` (stratified: every class, both strata) then
   the full set. **[CHAT] Human spot-check:** Track B session reviews 15 of
   the 50 against the rubric and records agreement with the judge.
4. **Drop policy:** implausible → dropped from v1 (kept in a `rejected/`
   sidecar with reasons); unsure → human-adjudicated in the spot-check
   session.
5. **Gate:** ≥90% of the 50-item stratified sample judged plausible
   (playbook T2.4); judge model + family documented in the manifest;
   human-judge agreement reported.
6. STATUS: `T2.4 | done | B | judgements.jsonl + rubric (X% plausible on
   50-sample, judge=<model>, agreement=Y/15)`.

## T2.6 — Stratify, balance, split

**Branch** `track-b/T2.6` · **Files** `benchmark/splits.py` ·
**Output** `data/benchmark/v1-pre/{train,dev,test}.jsonl` + `distribution.md`

1. **Leakage rule (the one that matters):** split by **base-program group**,
   never by instance — all mutants of one base land in one split; a model must
   never see the same base in train and test. CardDemo-locus instances group
   by source program.
2. **Real-curated placement:** the 21-instance seed goes **entirely to test**
   — train/dev are synthetic-only. The benchmark's credibility rests on real
   instances being unseen; say this in `distribution.md`.
3. **Pair atomicity:** T6 pairs are indivisible — both sides in the same
   split (test, given rule 2 puts all real pairs there; synthetic pairs from
   ×-variant bases follow their base group).
4. **Balance:** after judging drops, rebalance class proportions across
   splits by group-preserving assignment (greedy bin-fill is fine); report
   per-split × per-class × per-stratum counts in `distribution.md`, including
   any cell with n < 10 flagged as CI-fragile (feeds the T5.4 bootstrap
   design).
5. **Gate** `tests/test_splits.py`: zero base-group overlap between splits;
   all real_curated in test; every pair intact; distribution table
   regenerates deterministically.
6. STATUS: `T2.6 | done | B | data/benchmark/v1-pre/ splits + distribution.md`
   — and FLAGS → Track C: **M2 CLOSED**, synthetic v1 + seed ready for T4.1;
   include the distribution table path and any CI-fragile cells.
