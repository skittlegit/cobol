## Metric semantics

1. **T1 detection:** drift is positive, D7 is negative. Abstention is a miss at
   full coverage and excluded only from the separately reported answered set.
2. **T2 localization:** accuracy@1/@3 for program, paragraph, and line plus
   typed `(program,file,line)` overlap. D2 lines are insertion points.
3. **T3 classification:** macro-F1 over all seven classes; abstention supplies
   no predicted class.
4. **T4 faithfulness:** citation correctness is exact
   `(doc,clause_id,version,effective_date)` equality. Code-fact correctness
   requires a matching typed locus/line fact and rationale support recorded by
   the trajectory evaluator. Report only their conjunction, per verification
   tier.
5. **T5 migration:** accept optional behavioral outcomes but do not fabricate
   them when migration is absent.
6. **T6 versioned judgment:** pair by byte-equivalent `code_locus`; pair members
   must differ in clause version/effective date and correct verdict. Report
   paired accuracy, pair count, and exact binomial interval.

Every applicable metric is reported overall, per drift class, and for local
versus interprocedural strata. Empty cells are `null`, never zero.

## Statistical rules

- Fixed-seed paired bootstrap 95% intervals for headline values/deltas.
- Paired randomization test for system comparisons.
- Exact binomial interval for T6.
- At least 10,000 resamples for final M4 evidence; unit tests use fewer.

## Gates

Hand-computed fixtures cover perfect, wrong, abstaining, duplicate-line,
interprocedural, D2 insertion-point, tier-separated, and T6 paired cases.
Metrics must be invariant to input order and reject duplicate instance IDs.

## Completion

artifact is `ready-for-live-run`.
